标准输入输出

fgets函数

fgets（short for “file gets”）是C语言标准库中的一个输入函数，它用于从文件或标准输入（通常是键盘输入）读取一行文本，并将其存储到一个字符数组中。fgets的基本用法如下：

  • str：用于存储读取文本的字符数组的指针。这个字符数组必须足够大以容纳要读取的文本，通常会预分配一个足够大的数组来存储输入。
  • size：要**读取的最大字符数（包括终止null字符）(如果size为10，那么实际上从终端读取的字符的数量最多为9个字符，因为还有一个字符为 ‘\0’ )**。这个参数是为了避免溢出，通常设置为字符数组的大小。
  • stream：要从中读取文本的文件流。通常，您可以使用stdin来表示标准输入（键盘输入），或者使用一个文件指针表示打开的文件。

fgets函数会读取指定的字符数或者直到遇到换行符（’\n’）为止，将读取的文本存储在str指向的字符数组中，并在读取完毕后自动在末尾添加一个null终止符（’\0’），以确保字符串以null结尾。如果达到指定的字符数或者遇到换行符，fgets会停止读取。

strtok函数

strtok（short for “string tokenize”）是C语言标准库中的一个字符串处理函数，它用于将一个字符串分割成多个子字符串，通常是根据指定的分隔符来进行分割。strtok的基本用法如下：

  • str：要分割的字符串的指针。在第一次调用strtok时，将传递要分割的字符串，之后的调用应传递NULL。在后续调用中，strtok会继续从上一次停止的地方开始分割。
  • delimiters：包含分隔符的字符串，用于指定在哪些字符处分割字符串。strtok会根据delimiters中的字符来查找分割点。

strtok函数会返回分割出来的子字符串，并将内部状态更新以指向下一个分割点。如果没有找到分隔符，或者已经分割完字符串，strtok会返回NULL，表示没有更多的子字符串。

sscanf函数

sscanf 函数是C语言标准库中的一个函数，它用于从字符串中按照指定格式解析数据并将其存储到变量中。sscanf 的名称中的 “ss” 表示 “string scan”，它是与 scanf 相关的函数，用于从字符串而不是标准输入（键盘）读取数据。

sscanf 函数的原型如下：

  • str：要解析的输入字符串。
  • format：指定解析字符串的格式的字符串，类似于 scanf 函数中的格式字符串。
  • ...：可变数量的参数，用于接收解析后的数据，与 format 字符串中的格式说明符相匹配。

sscanf 函数按照 format 字符串的格式说明符来解析 str 中的数据，并将解析后的数据存储在后续的参数中。通常，你需要在 format 字符串中使用格式说明符，如 %d（整数）、%f（浮点数）、%s（字符串）等来指定要解析的数据类型和格式。

sprintf函数

sprintf 函数是C语言标准库中的一个函数，用于将格式化的数据写入字符串。与 printf 不同，sprintf 不是将输出发送到标准输出（通常是屏幕），而是将格式化的输出存储到一个字符串中。

sprintf 的原型如下：

  • str：指向目标字符串的指针，用于存储格式化后的数据。
  • format：格式字符串，类似于 printf 中的格式字符串。
  • ...：可变数量的参数，与 format 字符串中的格式说明符相匹配，用于生成输出。

sprintf 函数将格式化后的数据写入目标字符串 str 中，返回写入的字符数（不包括 null 终止符 \0）。目标字符串必须具有足够的空间来容纳生成的格式化数据，否则可能导致缓冲区溢出。

assert函数

assert 是C语言标准库中的一个宏，用于在程序中添加断言（assertion）。断言是一种用于检查程序是否满足某个条件的方法，如果条件不满足，则触发一个运行时错误，并终止程序的执行。assert 通常用于调试和验证代码，以确保程序的假设和预期是正确的。

assert 宏的原型如下：

  • expression：一个表达式，通常是一个条件表达式。如果该条件表达式的值为假（即0），assert 会触发一个运行时错误。

工作原理：

  1. 当程序执行到 assert(expression); 时，expression 将会被计算。
  2. 如果 expression 的值为真（非零），则程序继续正常执行，assert 不会做任何事情。
  3. 如果 expression 的值为假（0），assert 会触发一个运行时错误，并在标准错误流中打印一条出错信息，通常包括触发断言的文件名和行号。
  4. 然后，程序会终止执行。这有助于快速发现和修复代码中的问题。

stdarg.h头文件

stdarg.h 是C语言标准库中的一个头文件，用于支持可变数量的参数函数。这个头文件提供了一些宏和函数，用于在函数中处理不定数量的参数，例如printf和scanf等标准库函数。

stdarg.h 中最常用的宏和函数包括以下几个：

  1. va_list：va_list 是一个数据类型，用于声明一个参数列表的对象。通常，你会在函数中定义一个 va_list 类型的变量来处理可变数量的参数。
  2. va_start：va_start 宏用于初始化 va_list 对象，以便开始访问可变数量的参数。它需要两个参数：va_list 对象和最后一个固定参数的名字。
  3. va_arg：va_arg 宏用于获取可变数量参数列表中的下一个参数的值，并指定参数的数据类型。它需要两个参数：va_list 对象和参数的数据类型。
  4. va_end：va_end 宏用于结束可变数量参数列表的处理。它需要一个参数，即 va_list 对象。

char[]和char*

用法区别

在C语言中，char[] 和 char* 都与字符串处理有关，但它们在本质和用法上有一些重要的区别。

  1. char[]（字符数组）：

    • char[] 是一个字符数组，用于存储一串字符。它通常用于定义和存储字符串。
    • char[] 是一个固定大小的数组，必须在声明时指定其大小，例如 char str[20];。
    • 字符数组可以被修改，可以通过索引来访问和更改其元素。
    • 字符数组的大小在编译时确定，因此无法动态更改其大小。

  1. char*（字符指针）：
  • char* 是一个指向字符的指针，通常用于表示字符串。
  • char* 可以指向字符数组的首字符，也可以指向字符串常量。
  • 字符指针本身的大小是固定的，通常是4或8字节（取决于体系结构）。
  • 字符指针本身不能直接修改字符串的内容，但可以通过指针来访问和修改字符串的内容。

重要区别和用法：

  • 字符数组是一个完整的数据结构，存储在内存中，并且可以动态修改。你可以使用数组下标来访问和修改数组的元素。
  • 字符指针通常用于指向字符串的首字符，但它们本身并不存储字符串的内容。字符串通常以null终止符（\0）结尾，这使得字符指针可以用于迭代访问字符串中的字符。
  • 字符指针通常用于处理字符串，而字符数组通常用于存储字符串。

自动变量

在C语言中，自动变量（也称为局部变量）通常保存在栈上。栈是一种用于管理函数调用和局部变量的内存区域，具有后进先出（LIFO）的特性。

以下是有关自动变量在栈上存储的一些关键点：

  1. 自动存储期：自动变量具有自动存储期，它们的生命周期与其所在的块或函数的执行周期相对应。当函数被调用时，自动变量分配内存，当函数返回时，这些变量的内存被释放。
  2. 栈上分配：自动变量通常在栈上分配内存。每次函数调用时，栈上会为这些变量分配内存空间，当函数返回时，栈上的内存被回收，自动变量也就不再存在。
  3. 速度快：由于栈上分配和回收内存非常快，因此自动变量的访问速度通常很快。这使得它们成为存储临时数据和局部计算结果的理想选择。
  4. 局部作用域：自动变量的作用域限制在定义它们的块或函数内部。它们对于外部作用域是不可见的，这种封闭性有助于防止命名冲突和隔离变量的作用域。

随笔

  1. 数组的下标运算符：p[i] 本质上是 *(p+i)，因此p[i]也可以写成i[p]
  2. char[] 和 char* 在作为函数的参数的时候，并没有本质的区别
  3. 在C语言中，函数的参数通常是从右往左（从最后一个参数开始到第一个参数）被压入函数调用栈（stack）的。这意味着最后一个参数先被压入栈，然后是倒数第二个参数，以此类推，直到第一个参数。当你调用一个函数时，编译器会按照这个顺序将函数的参数压入栈中。这是为了支持可变数量的参数函数，例如printf和scanf，因为这些函数需要知道参数的数量和类型。在函数内部，参数从栈中依照相反的顺序被访问，也就是从左往右，从第一个参数到最后一个参数。函数可以使用这些参数来执行操作，然后返回结果。
