xv6实验记录

安装vmware + ubuntu + qemu + xv6

vmare + ubuntu

vmware之前已经安装过了，因此直接使用就可以。

然后去找一个ubuntu的镜像，找到国内的镜像，我找的是清华的镜像。下载完镜像之后，直接在操作系统上面安装ubuntu操作系统。

出现的问题：

  1. 由于当时电脑在安装ubuntu操作系统的时候，到最后卡死了，因此可能后面有一些配置啥的没装好，后来启动open-ssh服务端的时候出现了一些问题，从记：linux的ssh开启失败（Active: inactive (dead)） - Ying501 - 博客园 (cnblogs.com)找到了解决方案
  2. 可能是上述的问题导致的，后来发现xshell不能连接root用户，只能连接zby普通用户，从以下的博客找到解决方案，应该是sshd_config文件出现的配置问题。[xshell 使用root 连接ubuntu server - lick - 博客园 (cnblogs.com)](https://www.cnblogs.com/lick468/p/10744502.html#:~:text=xshell 使用root 连接ubuntu server 1 第一步 ：先使用账号登录 2,4 第四步：重启ssh服务 service ssh restart 5 第五步：重新连接，选择属性，使用root 账号登录)
  3. 自己电脑的虚拟机开启的时候，如果长时间不操作，会出现黑屏的问题，应该是windows服务的配置问题，需要重新关闭服务，重新开机，再打开服务，再重新开机即可，具体见如下博客。虚拟机黑屏解决方法_虚拟机黑屏怎么解决_一拳十个锵锵怪的博客-CSDN博客

3个练手的小程序

首先解释一下专业名词

QEMU（Quick Emulator）是一款开源的虚拟机监控器（VMM）和模拟器，它允许你在一个操作系统中模拟运行另一个操作系统或硬件平台。

xv6是一个简单的类Unix操作系统，它是由麻省理工学院开发的，并用于教授操作系统课程。xv6旨在帮助学生理解操作系统的基本概念和原理。

sleep

在xv6操作系统中，sleep函数用于让一个进程休眠一段指定的时间，然后再恢复执行。这对于实现进程调度和时间控制非常有用。

  1. 编写sleep.c，并将该文件拷贝到xv6源码目录下user/下
  2. 在xv6 源码根目录下的 Makefile 文件里，找到关键字UPROGS=\ ，在其下面列出的多个以&U开头的字符串的下方，添加一行&U/_sleep\，并保存
  3. 回到 xv6 的源码目录的根目录下，输入make qemu，以让 sleep.c 得以编译并
    在 xv6 内部形成可执行程序；
  4. 顺利进入到 xv6 的终端后，输入“./sleep 5”;

pingpong

这个练习程序叫做 “pingpong”，它是一个经典的进程间通信示例。程序的运行逻辑如下：

  1. 父进程使用 fork() 系统调用创建一个新的子进程。
  2. 父进程和子进程之间通过管道 pipe 建立了一个通信通道。
  3. 父进程向子进程发送一条消息（”ping”）。
  4. 子进程接收到消息后，回复父进程一条消息（”pong”）。
  5. 完成上述通信后，父进程和子进程都退出。

实现之前，需要了解一些系统调用，由于xv6是一个基于Unix的操作系统，因此它的一些系统调用与类Unix系统（如Linux）非常相似，都是基于POSIX标准的。POSIX（Portable Operating System Interface for Unix）是一组定义了Unix-like操作系统接口的标准。

思路：首先创建管道，然后创建子进程，子进程先读取父进程传递的数据pang，然后向父进程传递数据ping，父进程先向子进程传递数据pang，然后读取子进程传递的数据ping

primes

这个程序的目标是在终端输出2到35之间的所有质数。为了实现这个目标，程序采取了以下步骤：

  1. 父进程 A 负责生成数字范围[2-35]中的所有数字，包括2和35。
  2. 父进程 A 使用管道与其子进程 B 进行通信，将生成的数字传递给 B。
  3. 子进程 B 负责筛选这些数字，并打印出第一个质数2。
  4. 子进程 B 使用管道将剩余的数字传递给其子进程 C。
  5. 子进程 C 打印出第二个质数3。
  6. 重复步骤4和步骤5的过程，直到程序完成，在终端输出2到35之间的所有质数。

思路：父进程A生成数字，然后创建管道，创建子进程，并将数据传递给子进程，子进程读取数据之后，找到第一个质数，然后输出，再将剩下的数传递给下一个子进程。注意：父进程要等待子进程的结束再进行退出！（因此有一个比较好的方法来实现，就是用递归来做，因为递归做的时候，能够保证子进程所执行的函数先执行完毕！）
